Server Offline Auth

离线服务器强制注册/登录保护模组 ·

📖 概述

Server Offline Auth 是一款为 Minecraft Forge 1.20.1 服务器设计的强制登录保护模组。它要求玩家在进入服务器后必须注册或登录，未登录状态下无法移动、交互、聊天或破坏方块，有效防止盗号、偷窃和恶意破坏。同时支持 JSON 文件 和 MySQL 数据库 两种存储方式，并提供 本地 Token 免密登录 功能，让短时间内已登录过的用户无需重复输入密码。

✨ 功能特性

  • 🔒 强制注册/登录 – 未登录玩家被锁定在出生点，无法进行任何操作
  • 📁 双存储支持 – JSON 文件（默认）或 MySQL 数据库，可通过配置切换
  • 🎫 Token 自动登录 – 玩家登录后获得本地令牌，下次加入自动验证，实现“记住我”
  • ⚙️ 灵活配置 – 登录超时、BCrypt 强度、消息开关等均可自定义
  • 👑 管理员指令 – 强制登录/登出玩家，方便管理
  • 🌍 多语言支持 – 内置中文和英文，自动适配客户端语言
  • 💾 数据迁移工具 – 配置文件设置迁移目标，自动将 JSON 数据迁移到 MySQL（或反向）
  • 🛡️ 密码安全 – 使用 BCrypt 哈希存储，工作因子可调

⚙️ 安装方法

服务端安装

  1. 下载模组 JAR 文件：server-offline-auth-0.1.0-beta-1.20.1.jar
  2. 将 JAR 放入服务器的 mods/ 文件夹
  3. 重启服务器，模组会自动生成配置文件 config/server_offline_auth-common.toml
  4. 根据需求修改配置（详见配置说明）
  5. 再次重启服务器使配置生效

客户端安装

  1. 将模组 JAR 放入客户端的 mods/ 文件夹
  2. 启动游戏，连接服务器即可

⌨️ 命令列表

命令权限说明
/register <密码> <确认密码>玩家注册新账号，成功后自动登录并生成 Token
/login <密码>玩家使用密码登录，成功后自动登录并生成 Token
/logout玩家主动登出，回到未登录限制状态
/changepassword <旧密码> <新密码>玩家修改密码，Token 保持不变
/authtoken generate玩家手动生成新的 Token（覆盖旧 Token）
/authtoken clear玩家清除本地的 Token，下次需要手动登录
/serverofflineauth admin login <玩家>OP强制登录指定玩家（用于紧急情况）
/serverofflineauth admin logout <玩家>OP强制登出指定玩家

🛠️ 配置说明

配置文件位于 config/server_offline_auth-common.toml，主要配置项如下：

[general]
    # 登录超时（秒），0=禁用
    loginTimeout = 30
    # 盐值长度（4-30）
    bcryptWorkFactor = 10
    # JSON 数据文件名
    dataFileName = "server_offline_auth_data.json"
    # 加入时是否显示注册/登录提示
    joinMessageEnabled = true
    # Token 有效天数（0=永不过期）
    tokenExpiryDays = 30

[storage]
    # 存储类型: "json" 或 "mysql"
    storageType = "json"
    # MySQL 连接参数（当 storageType = "mysql" 时生效）
    mysqlHost = "localhost"
    mysqlPort = 3306
    mysqlDatabase = "ServerOfflineAuth"
    mysqlUsername = "root"
    mysqlPassword = "password"
    mysqlTable = "auth_players"

[migration]
    # 设置此项以执行数据迁移（如 "mysql" 或 "json"）。迁移成功后此项将自动清空，且 storageType 会被更新。
    migrateTo = ""

修改后重启服务器生效。

🗄️ 存储切换与数据迁移

模组支持在 JSON 和 MySQL 之间切换，并提供一键迁移功能。

  • 切换到 MySQL：先配置好 MySQL 连接信息，然后在 migration.migrateTo 填写 "mysql"，重启服务器。模组会自动将 JSON 中的所有玩家数据导入 MySQL，成功后自动将 storageType 改为 "mysql" 并清空 migrateTo。
  • 切换回 JSON：类似地，设置 migrateTo = "json" 重启，数据将从 MySQL 迁移到 JSON 文件。
  • 手动迁移：如果不希望自动切换，可以直接修改 storageType 并重启（不会迁移数据，会导致旧数据不可见）。
⚠️ 注意：迁移操作会覆盖目标存储中的现有数据，请提前备份。建议在服务器无玩家时进行。

🎫 Token 自动登录

Token 是一种免密登录机制，类似于“记住我”。工作流程如下：

  1. 玩家首次通过 /register 或 /login 成功登录后，服务器生成一个唯一 Token 并发送给客户端。
  2. 客户端将 Token 保存在 config/server_offline_auth_token.json 文件中（加密存储，但仅防篡改）。
  3. 下次玩家加入服务器时，客户端自动将 Token 发送给服务端验证。
  4. 服务端验证 Token 有效且未过期，则直接登录玩家，无需输入密码。

玩家可通过 /authtoken generate 手动刷新 Token，或 /authtoken clear 清除本地 Token（禁用自动登录）。

👑 管理员指令

管理员（OP）可以使用以下指令强制干预玩家状态：

  • /serverofflineauth admin login <玩家名> – 强制使指定玩家登录，恢复其上次的游戏模式及位置。
  • /serverofflineauth admin logout <玩家名> – 强制登出玩家，将其设为冒险模式并锁定。

这些指令在玩家卡登录或需要紧急处理时非常有用。

❓ 常见问题

问：Token 自动登录时提示“请先登录”，然后马上又显示“自动登录成功”？
答：这是正常现象。玩家加入时服务器先检测到未登录，会短暂提示；随后客户端发送 Token，服务器验证通过后立即自动登录。下个版本会优化消息顺序。
问：为什么我切换了存储类型，数据丢失了？
答：直接修改 storageType 不会触发数据迁移。请使用 migrateTo 配置项进行迁移，或手动复制数据文件。
问：MySQL 连接失败怎么办？
答：请检查 MySQL 服务是否启动、配置中的主机/端口/数据库/用户名/密码是否正确，以及 MySQL 驱动是否已放入 libs 文件夹并配置 minecraftLibrary。详细步骤见模组文档。
